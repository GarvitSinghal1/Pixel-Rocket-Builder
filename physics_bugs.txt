PIXEL ROCKET BUILDER - PHYSICS BUG REPORT
========================================

Ref: SpaceSim Implementation Audit
Priority: High (Physics/Simulation Accuracy)

--------------------------------------------------------------------------------
1. [PHYSICS] Gravity Constant Inconsistency
--------------------------------------------------------------------------------
- Location: js/physics.js vs js/planets.js
- Description: physics.js uses 9.8 or 9.81 in various local functions (e.g., calculateTWR), 
  while planets.js defines Earth surface gravity as 9.80665. 
- Impact: Slight errors in TWR and Delta-V telemetry.

--------------------------------------------------------------------------------
2. [LOGIC] Cross-Stage Fuel Draining
--------------------------------------------------------------------------------
- Location: js/physics.js (consumeFuelAndGetThrust)
- Description: The engine logic gathers ALL connected fuel tanks and drains them 
  equally. It does NOT check if a tank is separated by a decoupler.
- Impact: Engines drain fuel from upper stages prematurely, effectively giving 
  first-stage engines access to the entire rocket's fuel pool.

--------------------------------------------------------------------------------
3. [PHYSICS] Hardcoded Vacuum ISP
--------------------------------------------------------------------------------
- Location: js/physics.js (calculateAverageISP)
- Description: Vacuum ISP is hardcoded as exactly 15% better than sea level. 
  Specific 'isp' values in parts.js are often ignored or overridden by this 
  scaling factor in the main simulation loop.
- Impact: Unrealistic engine performance scaling across different altitudes.

--------------------------------------------------------------------------------
4. [PHYSICS] 1D Orbital Projection (The "Fake" Orbit)
--------------------------------------------------------------------------------
- Location: js/advanced.js (updateOrbitalState)
- Description: The simulation projects 1D vertical altitude/velocity into a 2D 
  state vector assuming zero horizontal velocity. 
- Impact: It is impossible to achieve a stable circular orbit; the rocket is 
  always on a purely radial trajectory. Orbital elements like Eccentricity 
  and True Anomaly are mathematically "pinned" to a radial plunge.

--------------------------------------------------------------------------------
5. [PHYSICS] Constant Radius Aerodynamic Heating
--------------------------------------------------------------------------------
- Location: js/physics.js (calculateAerodynamicHeating)
- Description: Uses a hardcoded noseRadius of 0.5m for all heat flux 
  calculations, regardless of whether a 1-tile or 3-tile fairing is used.
- Impact: Large fairings don't provide the realistic "blunt body" heat reduction 
  advantage.

--------------------------------------------------------------------------------
6. [UI] Advanced Telemetry Overflow
--------------------------------------------------------------------------------
- Location: html/simulator.html (#orbital-telemetry)
- Description: Advanced mode adds 9+ rows of labels. On small screens, the 
  telemetry panel overflows the flight controls.
- Impact: Critical flight controls (Throttle/Stage) can be obscured or pushed 
  off-screen.

--------------------------------------------------------------------------------
7. [LOGIC] Auto-Build Decoupler Shift Bug
--------------------------------------------------------------------------------
- Location: js/editor.js (autoAddDecouplers)
- Description: The function shifts ALL parts with Y > engineBottom further down. 
- Impact: Side boosters or radially attached parts that happen to be lower 
  than a central engine are shifted, breaking their side-attachments to the 
  rest of the rocket.

--------------------------------------------------------------------------------
8. [PHYSICS] Drag Coefficient Discontinuities
--------------------------------------------------------------------------------
- Location: js/physics.js (getDragCoefficient)
- Description: Sharp if/else transitions between Mach regimes (Subsonic/Transonic) 
  cause instant jumps in drag force.
- Impact: "Jerky" acceleration changes when crossing Mach 0.8 and Mach 1.0.

--------------------------------------------------------------------------------
9. [PHYSICS] Atmospheric Model Lapse Rate Fallback
--------------------------------------------------------------------------------
- Location: js/planets.js (getPlanetAtmosphere)
- Description: Fallback for planets without layer data uses Earth-like lapse 
  rates that don't cap temperature, potentially reaching unrealistic values.
- Impact: Inaccurate atmospheric simulation for Mars/Moon if layers aren't 
  defined strictly.

--------------------------------------------------------------------------------
10. [LOGIC] Staging Strut Fallback Performance
--------------------------------------------------------------------------------
- Location: js/physics.js (detectStages)
- Description: Fallback check for crossing struts is O(N^2) relative to part count.
- Impact: Possible frame-drops during staging events on very complex rockets.

--------------------------------------------------------------------------------
11. [UI] G-Force Indicator Saturation
--------------------------------------------------------------------------------
- Location: js/main.js (updateFlightData)
- Description: The G-indicator caps at 100% for 10G. 
- Impact: Fails to visualize the severity of structural-failure-level G-forces 
  (e.g., 50G+).

--------------------------------------------------------------------------------
12. [PHYSICS] Dynamic Pressure (Max Q) Limit Scaling
--------------------------------------------------------------------------------
- Location: js/physics.js (MAX_Q_LIMIT)
- Description: 35kPa is hardcoded. It doesn't scale with the structural parts used.
- Impact: A rocket with 10 struts is just as fragile as one with 1.

--------------------------------------------------------------------------------
13. [EDITOR] Disconnected Part Warning Threshold
--------------------------------------------------------------------------------
- Location: js/validation.js (validateRocketDesign)
- Description: Uses generic BFS starting from ALL engines. 
- Impact: If a player places a "dummy" engine on a disconnected pile of parts, 
  it passes validation even though it won't fly.
